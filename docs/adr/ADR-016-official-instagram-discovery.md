# ADR-016: Official Instagram professional-account discovery

- **Status:** Accepted and implemented for operator-triggered testing
- **Context:** Campaigns need a policy-aware Instagram lead source, while Meta does not provide arbitrary keyword, location or consumer-profile search and does not permit cold-DM initiation through the Send API.
- **Decision:** Use the official Instagram API with Facebook Login and Business Discovery for professional-profile URLs selected by the operator or already saved against campaign leads. Campaign hashtags open directly in Instagram as an assisted selection route; the local app does not scrape those results. Resolved profiles, public website enrichment and source evidence pass through the existing deduplication, scoring, product-matching and shortlist pipeline. Browser scraping, login automation and automatic messaging are prohibited.
- **Credentials:** App secret, user token and Page tokens are stored only in a current-user Windows DPAPI vault. The webview and SQLite receive status/account metadata but never secret values. OAuth uses a one-time state and a fixed loopback callback bound only to `127.0.0.1`.
- **Limitations:** Results are professional accounts only. Meta hashtag media does not expose a dependable author identity for automatic new-lead creation; consumer/private/unavailable profiles cannot be enriched; campaign location remains review context rather than an exact-radius assertion.
- **Permissions/features:** `pages_show_list`, `pages_read_engagement`, and `instagram_basic`. Publishing, messaging, comments, insights, advertising, business-administration and Public Content Access are not requested. App admins/developers/testers can test in development mode; serving unrelated accounts requires Meta review and any applicable business verification.
- **Consequences:** New Instagram candidates require one operator choice of profile URL.
