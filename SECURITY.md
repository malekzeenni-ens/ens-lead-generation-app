# Security policy

Do not commit API keys, OAuth tokens, passwords, provider payloads containing sensitive data, production databases or backups.

The API must bind only to `127.0.0.1` and require the per-session token. Report suspected vulnerabilities privately to the Etch ’N’ Shine business owner; do not place exploit details or personal data in public issue trackers.

External discovery, website fetching, AI and sending features stay disabled until their documented security gates pass. Production outreach also requires the recorded legal/privacy review described in `docs/compliance/CONTROL_REGISTER.md`.
